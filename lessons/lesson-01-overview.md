## 📖 Web サイトが表示される仕組み

### 日常の例え話で理解しよう

Webサイトが表示されるまでを、**レストランでの食事**に例えてみましょう：

```
あなた（ブラウザ）
    ↓ メニューを注文
サーバー（レストランの厨房）
    ↓ 料理を作って提供
あなた（ブラウザ）
    ↓ 料理を受け取って食べる
```

**実際のWebでは：**

```
1. あなたがブラウザでURLを入力
   例: https://camera-quiz.com

2. ブラウザがサーバーに「ページをください」とリクエスト

3. サーバーがHTMLファイルを返す

4. ブラウザがHTMLを読んで画面に表示

5. CSSで見た目を整える

6. JavaScriptで動きをつける
```

---

## 🏗️ Web アプリの3つの材料

Webアプリは主に**3つの言語**で作られています：

### 1️⃣ **HTML** - 骨組み
```
役割: ページの構造を作る
例え: 家の骨組み、設計図
```

**実際の例（index.html）：**
```html
<div id="root"></div>
```
↑ これは「ここにアプリを表示してね」という**箱**を用意しているだけ

---

### 2️⃣ **CSS** - 見た目
```
役割: 色、フォント、レイアウトを決める
例え: 壁紙、インテリア、装飾
```

**実際の例（index.css）：**
```css
.start-button {
  background: linear-gradient(135deg, #d4a853, #b8923f);
  padding: 18px 56px;
  color: #1a1a1a;
}
```
↑ ボタンの色、大きさ、余白を決めている

---

### 3️⃣ **JavaScript** - 動き
```
役割: クリックしたら何かが起こる、計算する
例え: 家電のスイッチ、エレベーター
```

**実際の例（StartScreen.tsx）：**
```javascript
const handleStart = () => {
  onStart(selectedDifficulty);
};
```
↑ ボタンを押したら「ゲームを開始する」という動作

## 🎨 フロントエンド vs バックエンド

### フロントエンド（このプロジェクト）
```
場所: あなたのブラウザ内
役割: 見た目と操作
例:
  - ボタンを押す
  - アニメーション
  - 画面の切り替え

使う言語: HTML, CSS, JavaScript
```

### バックエンド（このプロジェクトにはない）
```
場所: サーバー（遠くのコンピュータ）
役割: データの保存、計算、認証
例:
  - ユーザー登録
  - データベース保存
  - メール送信

使う言語: Python, Ruby, PHP, Node.jsなど
```

**Camera Quizの場合：**
- ✅ フロントエンドのみ
- ❌ バックエンドなし（データはブラウザ内で完結）
- 問題データは`questions.json`という**ファイル**に保存

## 📂 Camera Quiz の構造

このアプリがどう動いているか、流れを見てみましょう：

### ステップ1: 起動
```
index.html
  ↓ 読み込む
main.tsx
  ↓ Reactを起動
App.tsx
  ↓ どの画面を表示するか判断
```

### ステップ2: 画面の表示
```
状態によって画面を切り替え:

phase = 'start'    → StartScreen.tsx を表示
phase = 'playing'  → Question.tsx を表示
phase = 'result'   → Result.tsx を表示
```

### ステップ3: データの流れ
```
questions.json
  ↓ 問題データを読み込む
useQuiz.ts
  ↓ ゲームのロジック管理
各コンポーネント
  ↓ 画面に表示
```

## 🖥️ ブラウザの開発者ツールを使ってみよう

実際にWebサイトの裏側を見る方法があります！

### 開発者ツールを開く方法：

**Chrome / Edge：**
- Windows: `F12` または `Ctrl + Shift + I`
- Mac: `Cmd + Option + I`

**Firefox：**
- Windows: `F12` または `Ctrl + Shift + I`
- Mac: `Cmd + Option + I`

### 何ができる？

1. **Elements タブ**
   - HTMLの構造を見る
   - CSSのスタイルを確認
   - 試しに色を変えてみる（一時的）

2. **Console タブ**
   - JavaScriptのエラーを見る
   - console.logの出力を見る

3. **Network タブ**
   - どんなファイルを読み込んでいるか
   - 読み込み時間

### 🔍 実践：このページで試してみよう

1. このページ（GitHubのREADMEページ）で開発者ツールを開く
2. Elements タブを開く
3. 左上の矢印アイコンをクリック
4. ページ内の要素にカーソルを合わせる
5. クリックすると、そのHTMLコードが表示される！

## 💡 重要な概念: コンポーネント

Camera Quizは**3つの画面（コンポーネント）**でできています：

### 🏠 StartScreen（スタート画面）
```
役割: 難易度を選んでスタート
ファイル: src/components/StartScreen.tsx
表示内容:
  - タイトル
  - 難易度選択ボタン
  - スタートボタン
```

### ❓ Question（問題画面）
```
役割: 問題を表示して回答を受け付ける
ファイル: src/components/Question.tsx
表示内容:
  - 進捗バー
  - 問題文
  - 4つの選択肢
  - 解説（回答後）
```

### 🏆 Result（結果画面）
```
役割: スコアと評価を表示
ファイル: src/components/Result.tsx
表示内容:
  - スコア
  - パーセンテージ
  - 評価メッセージ
  - もう一度ボタン
```

**これがReactの「コンポーネント思考」です！**
大きなアプリを小さなパーツに分けて管理します。
